# Glossary: Coffee-ERP System Terms

## 1. Иерархия и Доступ

* **Бизнес (Business/Tenant):** Верхнеуровневый контейнер системы. Обеспечивает полную изоляцию данных; один "Бизнес" не имеет доступа к данным другого.
* **Предприятие (Enterprise):** Юридическое лицо внутри Бизнеса, используемое для группировки подразделений.
* **Подразделение (Unit):** Конечный объект системы (кофейня, склад или цех производства). Может совмещать несколько ролей одновременно.
* **Super-User (SU):** Провайдер всей платформы с God-mode доступом ко всем созданным Бизнесам.
* **Управляющий (Business Owner):** Пользователь с абсолютными правами внутри конкретного Бизнеса.
* **RBAC (Role-Based Access Control):** Ролевая модель доступа, определяющая полномочия (Админ, Менеджер, Бариста) в рамках конкретных Unit-ов.
* **Ownership Handshake ("Рукопожатие"):** Безопасный процесс передачи прав Владельца Бизнеса с периодом "заморозки" (Cooldown) 168 часов.

## 2. Склад и Производство

* **ТТК (Технико-технологическая карта):** Динамический движок рецепта, автоматически рассчитывающий себестоимость (COGS), маржу и КБЖУ.
* **MOQ (Minimum Order Quantity):** Квант заказа или минимальный шаг (например, упаковка из 12 шт), используемый при формировании заявок.
* **Scope (Глобальный vs Локальный):** Уровень видимости продукта. Глобальный — общие настройки для всей сети; Локальный — индивидуальные цены для конкретной точки.
* **Слепая ревизия (Blind Inventory):** Метод инвентаризации, при котором сотрудники вводят фактические остатки, не видя расчетных данных системы.
* **Декомпозиция (Разворот):** Функция в ревизии, позволяющая разложить сложный компонент (соус, заготовку) на базовые ингредиенты по формуле ТТК.

## 3. Персонал и Финансы

* **Z-отчет:** Итоговый документ кассовой смены, данные из которого вносятся вручную для расчета финансовой эффективности.
* **Food Cost (Фуд-кост):** Отношение себестоимости потраченных продуктов к выручке за период, выраженное в процентах.
* **Анти-зомби (Auto-Close):** Система защиты от бесконечных смен, автоматически закрывающая смену при превышении лимита времени.
* **Soft Delete:** Для объектов — хранение 1 год с уведомлением за месяц до удаления. Для сотрудников — вечное хранение персональных и финансовых данных.Payroll:** Модуль автоматического расчета выплат на основе подтвержденных часов из графика и логов смен.
* **Broken Ingredient:** Статус сырья, отключающий зависимые ТТК.
* **Deprecated Tab:** Специальный раздел в М2 для редактирования ТТК со «сломанными» компонентами.

## 4. Контроль и Качество

* **ХАССП (HACCP):** Система цифровых журналов для контроля температурного режима и санитарных норм.
* **Инцидент:** Любая внештатная ситуация (поломка, списание, жалоба), требующая реакции Управляющего.
* **SLA (Service Level Agreement):** Регламентированное время прибытия мастера или устранения поломки, указанное в карточке оборудования.
* **Emergency Hub (Карточка реагирования):** Интерфейс быстрого доступа к контактам сервисных служб при возникновении инцидента.
* **Комплаенс (Compliance):** Система автоматического контроля сроков действия медицинских книжек и допусков персонала.
