## 1. Назначение и Философия

Модуль предназначен для создания публичного интерфейса взаимодействия с гостем.
Система автоматически преобразует данные производства в Digital Menu.

- **Data-Driven Menu:** Меню собирает данные о составе, КБЖУ и аллергенах напрямую из техкарт Модуля 02.
- **Дизайн:** Публичное меню выполнено в стиле Glassmorphism (эффект стекла, блюр), оптимизировано под мобильные устройства.

## 2. Ключевые компоненты

### 2.1 Цифровое меню (Digital Menu)

- **Instant Safety Sync:** Блюдо исчезает из меню мгновенно,
если обязательный ингредиент помечен как «сломан» в Модуле 01.

- **Нутрициология:** Отображение честного КБЖУ, рассчитанного системой в реальном времени.
- **Multi-language:** Поддержка нескольких языков интерфейса и названий блюд.

### 2.2 QR-доступ (QR Generator)

Механизм адресации юнитов:

- **Unique Links:** Привязка QR к конкретному `Unit_ID`.
- **Dynamic Routing (Medium):** Роутинг гостя на актуальные акции в зависимости от времени (завтраки/ланчи).

### 2.3 Маркетинговый бюллетень (Marketing Engine)

Инструмент коммуникации:

- **Promo Banners:** Размещение временных баннеров с привязкой к календарю.
- **Feedback Loop:** Интеграция с Модулем 08 для сбора отзывов и жалоб.

## 3. Техническая реализация

- **Performance:** Максимально легкий UI на SCSS без лишних библиотек.
- **Caching Logic:** Данные меню кэшируются на уровне сервера. Инвалидация происходит по триггеру из Модулей 01/02.
- **Unit Switcher Loader:** Плавная анимация загрузки при переключении между меню разных точек.
